# v0.2.0 Upgrade Guide

Draft sections from v1 feature tasks. t13-06 merges these into the final customer-facing guide.

## Configurable timeouts

v1 adds module-managed `timeouts` blocks defaulting to `"30m"` on Atlas and GCP resources where the provider supports them. The first plan after upgrade may show in-place timeout block additions. These updates are not destructive.

Set `timeouts = null` to skip all module-managed timeout blocks and keep provider defaults. Use this for zero-diff upgrades from v0.x or after `terraform import`:

```hcl
module "atlas_gcp" {
  source   = "..."
  timeouts = null
  # ...
}
```

Remove or replace `timeouts = null` later if you want the module 30m defaults.
